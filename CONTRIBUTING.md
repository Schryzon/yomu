# Contributing to yōmu!


First off, thank you for considering contributing to yōmu! It's people like you who make this tool better for language learners around the world.

## How Can I Contribute?


### Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/Schryzon/yomu/issues).
- Describe the bug and provide steps to reproduce it.
- Include information about your browser and OS.

### Suggesting Enhancements
- Open a new issue with the tag `enhancement`.
- Explain why the feature would be useful and how it should work.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the project's style (see below).

## Coding Style
- **Readability > Everything**: Spaced out, consistent, easy to scan.
- **Snake_case**: Use `snake_case` for all backend and system-level naming.
- **Flat Logic**: Avoid deep nesting. Use early returns and clear branching.
- **Optimistic Error Handling**: Let it crash predictably rather than swallowing errors silently.
- **Standardized Formatting**: Follow the existing patterns in the codebase.
